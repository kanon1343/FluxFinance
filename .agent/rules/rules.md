# FluxFinance プロジェクトルール

本プロジェクト（**FluxFinance**）において、Google Antigravity エージェントが遵守すべき全体的な指示およびプロジェクト固有のルールです。

## 1. グローバルなエージェント指示
- **言語要件:**
  - 思考プロセス (Thoughts) は英語で行うこと。
  - ユーザーへの返答 (Responses) は必ず**日本語**に翻訳すること。
  - Artifacts（ドキュメントなど）は、ユーザーにレビューを求める前に**日本語**で作成・翻訳すること。
- **実行制限:**
  - タスク実行中、ステップ（Step）の数が**10**を超えた場合は、自律的な処理を一度停止すること。
  - その際、「完了した作業」「現在の状況の報告」「次のアクション」をユーザーに報告し、判断（承認）を仰ぐこと。

## 2. 技術スタックと環境
- **フレームワーク:** Next.js (App Router)
- **言語:** TypeScript
- **パッケージマネージャー:** pnpm
- **ツール:**
  - Linter & Formatter: **Biome** (明確な指示がない限り ESLint/Prettier は使用しないこと)。
  - ユニットテスト: **Vitest**
  - E2E テスト: **Playwright**
- **環境:**
  - 隔離された開発環境として VS Code DevContainer を使用する。
  - Playwright の E2E テストでは、隔離されたネットワーク構成が関わる場合がある。

## 3. 開発ガイドライン
- パッケージ管理のコマンドにはすべて `pnpm` を使用すること（例: `pnpm install`, `pnpm dev`, `pnpm add`）。
- コードが Biome のフォーマットおよび lint ルールを通過することを確認すること。コード変更時は `biome format` および `biome lint` を使用する。
- Playwright を使用した E2E テストについては、`playwright.config.ts` の既存の設定を参照すること。AI ベースの探索的テスト（モンキーテスト）が必要な場合は、ローカルの Ollama モデルと `.agent/skills/ollama-monkey-testing/SKILL.md` のアプローチを活用すること。
- DevContainer やネットワーク（特に LLM/Ollama 統合に関する部分）を設定する際は、ネットワーク隔離ルール（ローカルホスト通信のみに制限するなど）が遵守されるようにすること。

## 4. ドキュメント
- 新しいコア依存関係や環境設定が導入された場合は、`README.md` を更新すること。
- 明示的かつセマンティックな HTML 構造を使用し、UI コンポーネントにおけるアクセシビリティ (a11y) を優先すること。
