# [Feature] コンテキスト連動型 AI金融アシスタント（チャットUI）の導入

## 概要
ダッシュボード上に表示されている現在の株価データやヒストリカルデータをコンテキスト（文脈）として理解し、ユーザーからの自然言語での質問に回答するAIチャットアシスタント機能を導入する。

## 背景・解決したい課題
ユーザーが特定の銘柄の動向や市場全体のトレンドについて疑問を持った際、現在のダッシュボードだけでは回答を得られず、外部の検索エンジンやチャットAIに切り替えて質問を打ち直す必要がある。また、外部AIはユーザーが現在見ているリアルタイムな株価情報を知らないため、正確な回答が得られないことが多い。

## ユーザーストーリー
* **As a (〜として)**: ポートフォリオの状況についてアドバイスが欲しいユーザー
* **I want to (〜したい)**: 現在のダッシュボードの情報を理解しているAIに対して、画面右下のチャットから直接質問したい
* **So that (なぜなら)**: ツールを切り替えることなく、具体的な銘柄の動向や市場のインサイトを対話形式で素早く得たいから

## 技術的アプローチ・詳細
1. **ライブラリ導入:** `ai` (Vercel AI SDK) を導入し、ストリーミングレスポンスに対応したバックエンド・フロントエンドを構築する。
2. **チャットコンポーネント:** 画面右下にトグル開閉式のチャットUI（`ChatAssistant.tsx`）を実装する。
3. **コンテキストの注入:** ユーザーがメッセージを送信する際、現在の `stocks` ステート（銘柄名、現在価格、変動率など）をバックグラウンドでシステムプロンプトとしてLLMに渡す（RAGの簡易版）。
4. **APIルートの作成:** `src/app/api/chat/route.ts` を作成し、LLM（Ollamaまたは外部API）へのストリーミング通信を処理する。

## 受け入れ条件 (Acceptance Criteria)
- [ ] 画面の右下にフローティング形式のチャットボタンが表示され、クリックでチャットウィンドウが開閉すること。
- [ ] ユーザーがチャットで質問を入力し、送信できること。
- [ ] Vercel AI SDKを用いたストリーミング形式で、LLMからの回答が徐々に表示されること。
- [ ] LLMの回答が、現在のダッシュボードに表示されている最新の株価データを踏まえた内容になっていること（例：「Appleの今の株価は？」に対して正確に答えられること）。
- [ ] 対話の履歴（メッセージ）がコンポーネントのステートに保持され、連続した会話が成立すること。
